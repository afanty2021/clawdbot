# Gateway Status 命令子模块

> 更新时间：2026-02-20

本模块实现 `openclaw gateway status` 命令，检查网关服务状态和可达性。

## 模块概览

```
src/commands/gateway-status/
└── helpers.ts    # 状态检查辅助函数
```

主要实现在 `src/commands/gateway-status.ts`。

## 核心功能

### 1. 命令实现 (`gateway-status.ts`)

**导出**: `gatewayStatusCommand(opts, runtime)`

**命令**: `openclaw gateway status`

**选项**:
| 选项 | 类型 | 说明 |
|------|------|------|
| `--url` | string | 显式指定网关 URL |
| `--token` | string | RPC 认证令牌 |
| `--password` | string | 密码认证 |
| `--timeout` | number | 超时时间（毫秒） |
| `--ssh` | string | SSH 隧道目标 |
| `--ssh-identity` | string | SSH 身份文件 |
| `--ssh-auto` | boolean | 自动 SSH |
| `--json` | boolean | JSON 输出 |

### 2. 目标解析 (`helpers.ts`)

**导出**: `resolveTargets(cfg, explicitUrl)`

**目标类型**:
- **explicit**: 显式指定的 URL
- **configRemote**: 配置中的远程 URL
- **localLoopback**: 本地回环 (127.0.0.1)
- **sshTunnel**: SSH 隧道

```typescript
type GatewayStatusTarget = {
  id: string;
  kind: TargetKind;
  url: string;
  active: boolean;
  tunnel?: {
    kind: "ssh";
    target: string;
    localPort: number;
    remotePort: number;
    pid: number | null;
  };
};
```

### 3. 探测预算 (`helpers.ts`)

**导出**: `resolveProbeBudgetMs(overallMs, kind)`

根据目标类型分配探测时间：
- **localLoopback**: 最多 800ms
- **sshTunnel**: 最多 2000ms
- **其他**: 最多 1500ms

### 4. 认证解析 (`helpers.ts`)

**导出**: `resolveAuthForTarget(cfg, target, overrides)`

**认证优先级**:
1. 命令行覆盖 (`--token`, `--password`)
2. 远程配置 (remote token/password)
3. 环境变量 (`OPENCLAW_GATEWAY_TOKEN/PASSWORD`)
4. 配置文件 (gateway.auth)

### 5. 配置摘要 (`helpers.ts`)

**导出**: `extractConfigSummary(snapshot)`

**提取配置信息**:
- 配置文件路径和有效性
- 网关模式（local/remote）
- 监听地址和端口
- 认证模式
- 远程 URL 和认证
- Tailscale 模式
- 发现设置

### 6. 网络提示 (`helpers.ts`)

**导出**: `buildNetworkHints(cfg)`

**构建网络信息**:
- 本地回环 URL
- Tailscale URL
- 主 Tailnet IP

### 7. 格式化 (`helpers.ts`)

**导出**:
- `renderTargetHeader(target, rich)` - 渲染目标标题
- `renderProbeSummaryLine(probe, rich)` - 渲染探测结果
- `parseTimeoutMs(raw, fallback)` - 解析超时
- `sanitizeSshTarget(value)` - 清理 SSH 目标

## 状态检查流程

```
1. 解析目标
   ├─ 显式 URL
   ├─ 配置远程 URL
   └─ 本地回环

2. 服务发现
   ├─ Bonjour/mDNS
   └─ 广域 DNS

3. SSH 隧道（可选）
   ├─ 解析目标
   ├─ 启动端口转发
   └─ 处理错误

4. 探测网关
   ├─ 连接检查
   ├─ 认证检查
   └─ RPC 调用

5. 汇总结果
   ├─ 连接状态
   ├─ 延迟信息
   └─ 错误详情
```

## SSH 隧道

**启动流程**:
1. 解析 SSH 目标（用户@主机）
2. 解析身份文件（~/.ssh/id_rsa）
3. 启动端口转发（`ssh -L ...`）
4. 记录进程 ID
5. 使用完毕后清理

**配置**:
```yaml
gateway:
  remote:
    sshTarget: "user@remote-host"
    sshIdentity: "~/.ssh/id_rsa"
```
